# Linux Basics — Day 1

## Commands I learned today

- `cd` – change directory
- `ls -la` – list all files with detailed permissions
- `grep` – search for specific words inside files
- `mkdir` – create a new directory
- `touch` – create a new file
- `|` (pipe) – send output of one command as input to another

## What I understood

- How to navigate directories and create files/folders
- Using `grep` to search for specific words inside files
- Using pipe (`|`) to combine commands
- Linux permissions: **owner**, **groups**, and **others** – each has different read/write/execute permissions
- Searching for files and extracting only specific words from them

## What confused me

I was confused about who **groups** and **others** were.

Then I learned:

- **Groups** – If 10 developers are working on the same project, instead of giving each person individual permissions, they all inherit the group permissions.
- **Others** – Everyone else who is not the owner and not in that group.
